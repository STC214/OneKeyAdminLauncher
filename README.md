# OneKeyAdminLauncher

一个使用 Go + Win32 原生控件编写的 Windows 程序启动管理器。程序自身以管理员权限运行，用于一键以管理员权限启动、检测和关闭一组常用工具。

版本号在构建时按 `yyyyMMdd_HHmm` 自动生成，例如 `20260711_1430`。主窗口标题显示该版本号，Windows 文件属性的“详细信息”页面包含产品名称、文件说明、版本、公司、版权、原始文件名和应用备注。

## 功能

- 一键启动所有已启用的程序项，最多同时启动 6 个；普通文件路径不存在时直接汇总报错，多个或重叠批次的启动失败合并到一个消息框。
- 一键关闭所有已启用项对应的进程。
- 支持普通 `.exe`、Windows 快捷方式 `.lnk` 和 UWP 启动项。
- 支持手动从当前进程列表中绑定检测和关闭使用的进程名。
- 每秒自动刷新运行状态。
- 一键关闭会先向所有目标程序发送正常关闭消息，共享同一个等待期限；超时后再批量使用 Win32 API 强制结束仍存活的进程。
- 一键关闭会按进程名去重；枚举、关闭或强制结束失败时汇总显示具体错误。
- 启动和关闭辅助流程不弹出 cmd 窗口。
- 支持最小化到系统托盘。
- 自动保存程序列表、启用状态、UWP 标记、进程绑定和窗口位置/大小。
- 自动保存通过单一后台队列顺序写入；失败时提示用户，退出保存失败时可选择重试、返回程序或放弃修改退出。
- 图标嵌入 exe，覆盖标题栏、Windows 任务栏和系统托盘。
- 列表使用虚拟行控件池，程序项较多时只渲染可见行，滚动更稳且不易卡顿。
- 窗口最小宽度固定为当前屏幕宽度的一半，避免控件被压坏。

## UI 行为

- 主界面保持紧凑的深色 Win32 工具风格。
- 顶部工具栏固定显示，不随列表滚动。
- 程序列表超出窗口高度后使用右侧滚动条。
- 滚动时复用可见行控件，避免为所有配置项创建大量 Win32 子窗口。
- 窗口缩放时会擦除旧绘制区域，避免控件边框残影；滚动时使用轻量刷新，减少闪动。
- 最小化按钮会隐藏主窗口并进入托盘；双击托盘图标恢复。
- 托盘菜单“退出”和窗口关闭按钮共用同一套保存确认流程。

## 配置文件

运行时配置保存在 exe 同级目录：

```text
launcher_config.json
```

如果检测到旧版配置：

```text
data/launcher_config.json
```

程序启动时会自动迁移到 exe 同级目录。

## 构建

构建环境需要：

- Windows
- Go
- `windres`
- Python + Pillow

构建命令：

```powershell
cd F:\Project\04_Desktop_Dev_Utilities\OneKeyAdminOpenPro
.\scripts\build.ps1
```

构建脚本会自动执行测试并生成：

```text
dist/程序启动管理器.exe
```

仓库提交根目录的 `ICO.ico`，保证全新克隆后可以直接构建。如果本地另有更新的 `ICO.png`，构建脚本会先将其转换为多尺寸 `ICO.ico`。随后通过 `app.rc` 生成 `app.syso` 并嵌入最终 exe；源 PNG、`app.syso` 和 `dist/` 不提交到仓库。

## 项目结构

```text
cmd/program-launch-manager/    程序入口、manifest 和资源脚本
internal/config/               配置加载、保存和迁移
internal/process/              启动、进程枚举和关闭逻辑
internal/winui/                Win32 UI
scripts/build.ps1              构建脚本
docs/                          规格和构建说明
```

## 备注

程序 manifest 使用 `requireAdministrator`。双击启动时会触发 UAC；如果拒绝管理员权限，程序会直接退出。未嵌入 manifest 的调试构建会通过 `runas` 启动管理员子进程，原非管理员进程随后退出，避免出现双实例。
