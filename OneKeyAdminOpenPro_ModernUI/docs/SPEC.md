# 程序启动管理器 Go + Win32 重构规格

## 目标

在本项目目录内重构截图中的 `程序启动管理器`，使用 Go + Win32 原生控件实现一个可移植的 Windows 桌面工具。程序本身以管理员权限运行，用于批量以管理员权限启动、监控和关闭用户配置的程序。

## 核心功能

### 管理程序列表

主窗口显示一个可编辑程序列表。每一行代表一个启动项，包含：

- 启用复选框：决定该行是否参与“一键开启”和“一键关闭”。
- 程序路径输入框：保存 exe、lnk 或 UWP 启动目标。
- `浏览` 按钮：选择本地 exe/lnk 等程序路径。
- 状态/进程框：显示绑定的进程名或运行状态。
- `选择进程` 按钮：弹出当前进程列表，手动绑定用于检测和关闭的进程名。
- 删除按钮：移除当前行。

程序项数量超过窗口可见高度时，列表区域使用右侧滚动条。Modern UI 子项目的主列表滚动条为自绘细滚动条，支持滚轮、轨道点击和滑块拖动；`选择进程` 对话框里的进程列表仍使用原生列表滚动条。列表实现采用虚拟行控件池：窗口中只创建可见行数量附近的 Win32 子控件，滚动时复用这些控件并重新绑定到对应配置项，避免项目较多时创建大量不可见控件导致 UI 卡顿或闪动。

### 顶部操作

- `一键开启`：启动所有已启用的程序项。
- `一键关闭`：关闭所有已启用项对应的进程。
- `添加程序`：新增一条程序配置。
- `全选`：将所有程序项设为启用。
- `全不选`：将所有程序项设为未启用。
- `保存配置`：立即保存列表和窗口状态。

### 配置加载与保存

程序启动时自动读取配置文件。

配置记录每个程序项：

- `path`：启动目标路径。
- `process_name`：自动推断或兼容旧版的进程名。
- `selected_process`：用户手动绑定的进程名。
- `is_uwp`：是否为 UWP 启动项。
- `enabled`：是否启用。

重构版还需要保存：

- 主窗口位置。
- 主窗口大小。

配置文件默认放在 exe 同级目录的 `launcher_config.json`，保持便携且便于直接查看。若检测到旧版 `data/launcher_config.json`，启动时自动迁移到 exe 同级目录。

用户修改路径、启用状态、UWP 标记、进程绑定、添加/删除项目、批量全选/全不选或退出程序时都会自动保存配置；`保存配置` 按钮用于立即手动落盘确认。

自动保存使用单一后台队列严格按提交顺序写入，避免旧快照覆盖新配置。首次自动保存失败时提示用户；退出时保存失败可选择重试、返回程序或明确放弃未保存修改后退出。窗口关闭和托盘退出共用该流程。

### 管理员权限

UI 程序启动时必须请求管理员权限。

- 用户允许 UAC 后，主窗口正常打开。
- 用户拒绝 UAC 时，程序直接退出。
- 主程序以管理员权限启动后，列表中选中的程序也会由该管理员进程启动。
- 对未嵌入 manifest 的调试构建，非管理员进程通过 `runas` 启动子进程后必须立即退出，不能同时继续显示第二个非管理员窗口。

### 启动程序

支持启动：

- 普通 exe。
- Windows 快捷方式 `.lnk`。
- UWP 应用项。

普通文件使用 ShellExecute 启动。UWP 项使用 `shell:AppsFolder\...` 或保存的 UWP AppUserModelID 启动。

### 关闭程序

关闭顺序：

1. 根据绑定进程名查找进程。
2. 先尝试正常关闭窗口。
3. 等待短时间确认进程退出。
4. 如果仍未退出，再强制结束进程。

正常关闭优先使用主窗口关闭消息，并一次性通知所有目标程序、共享同一个等待期限；超时后再批量 kill 仍存活的进程，避免等待时间随程序数量线性累加。

批量关闭前按不区分大小写的进程名去重。进程枚举、窗口枚举、打开进程、强制终止或最终存活检查失败时，由后台任务汇总错误并返回主界面显示。

### 运行状态刷新

启动后每秒刷新一次所有行的运行状态。

- 正在运行：状态框显示绿色进程名。
- 未运行：状态框显示灰色 `未运行`。
- 手动绑定了进程名时，优先使用 `selected_process` 检测。
- 没有绑定时，优先从 `process_name` 或路径文件名推断。

### 选择进程

点击 `选择进程` 后弹出当前进程列表。

列表显示当前可见的进程名，用户选择后写入该行的 `selected_process`，用于后续状态检测和关闭。

### UWP 支持

新增或编辑项时允许保存 UWP 标记。

UWP 启动项使用 `is_uwp=true`，路径字段保存可启动的 UWP 标识或 shell URI。状态检测仍依赖用户选择的进程名。

### 托盘支持

支持最小化到托盘。

- 点击最小化时隐藏主窗口，并显示托盘图标。
- 双击托盘图标恢复窗口。
- 托盘菜单包含 `显示` 和 `退出`。
- 关闭按钮按正常退出处理，并保存配置。
- 进程选择窗口打开期间从托盘退出时，先关闭该对话框并将 `WM_QUIT` 保留给主消息循环，不能留下无窗口后台进程。
- 托盘图标添加失败时不隐藏窗口；Explorer 重启并广播 `TaskbarCreated` 后自动重建图标，重建失败则主动恢复主窗口。
- 托盘图标、标题栏图标和 Windows 任务栏图标使用同一个嵌入式高清 ico 资源。

### 窗口状态

程序退出时保存窗口大小和位置。

下次启动时恢复到上次位置。若上次位置不在当前屏幕可见区域，则回退到默认居中位置。

窗口最小宽度按当前所在屏幕宽度的一半计算，同时保留 Modern UI 布局所需的最低宽度。程序启动时若旧配置保存的窗口宽度小于该最小宽度，会自动提升。窗口最小高度保持为 `360`。

## UI 风格

保持截图中的紧凑深色 Win32 工具风格：

- 窗口标题：`程序启动管理器`。
- 深灰背景。
- 顶部彩色功能按钮。
- 每行高度固定，横向排列。
- 顶部工具栏固定显示，不随程序列表滚动。
- Modern UI 子项目使用更现代的深色调、Segoe UI 字体、扁平按钮和右侧固定对齐布局。
- 顶部操作和行内操作按钮使用彩色自绘样式；UWP 标记文字需要保持高对比度，避免融入深色背景。
- 标题栏使用 DWM 深色标题栏属性，保留系统原生窗口拖动、缩放和标题栏按钮行为。
- 运行进程名使用绿色文本。
- `未运行` 使用灰色文本。

滚动和窗口缩放使用不同的刷新策略：滚动时尽量不擦背景以减少闪动；窗口创建、缩放、添加和删除行时会擦除背景以避免旧控件边框残影。

该程序属于低频 UI 变动工具。空闲状态下应避免持续触发 UI 重绘、控件重排或无意义的 UI 线程消息；定时状态刷新在后台枚举进程，但只比较配置项关心的进程存活状态，只有这些存活状态发生变化时才通知 UI 更新。

## 非目标

首版不实现复杂分组、启动延迟队列、日志面板或云同步。功能重点是复刻截图工具的实际使用流程，并保证管理员启动、配置持久化和批量控制可靠。
